# DEMO_AUTH_AND_PRINT_SHOP_SCOPE.md

**Date**: 2026-05-25
**Purpose**: Clarify how demo sessions should talk about current EMS login, future POLSCI OAuth integration, and the planned print-shop lane.

---

## Demo Messaging

- Demo may show the existing EMS native login.
- POLSCI OAuth is planned for Faculty LAN integration and is not implemented yet.
- Print-shop access is a separate future or controlled-access lane.

---

## Demo Safety Rules

- Do not demo print-shop access as if it were a CMU user flow.
- Do not claim the POLSCI callback contract is complete.
- Do not imply that external partner auth is production-ready.
- If a print-shop page is shown, label it as an existing EMS role surface whose external auth ownership is still pending.

---

## Recommended Talk Track

1. EMS currently supports native EMS login.
2. Faculty LAN target has been identified.
3. POLSCI OAuth entry point has been observed.
4. Contract verification is still required before implementation.
5. Print-shop users will use a separate limited-access lane rather than being forced into CMU identity assumptions.
