# POST_DEMO_DECISION_MATRIX.md

**Date**: 2026-05-25  
**Purpose**: Structured options for leadership decision after stakeholder demo.

## Decision Options

**1. Continue Demo Polishing Only**  
When to choose: Feedback shows UX polish is the main blocker.  
Required evidence: More screenshots, edge-case fixes, full rehearsal.  
Owner: EMS team.  
Next action: 1–2 week polish sprint.  
Risk: Delays real progress on external blockers.

**2. Proceed to Faculty LAN Pilot Preparation**  
When to choose: Strong positive feedback + leadership wants deployment momentum.  
Required evidence: Laravel contract answers + PG target + backup evidence.  
Owner: IT + EMS + DPO.  
Next action: Send contract request immediately.  
Risk: High if contract remains unanswered.

**3. Prioritize Laravel Auth Contract**  
When to choose: Almost everyone agrees this is the #1 blocker.  
Required evidence: Written answers from Laravel owner + IT.  
Owner: EMS lead + Faculty IT liaison.  
Next action: Formal request package sent within 48 hours.  
Risk: Depends entirely on external response time.

**4. Prioritize UI Redesign**  
When to choose: Feedback repeatedly cites "looks dated" or "hard to use".  
Required evidence: Design handoff package already exists.  
Owner: Design + EMS team.  
Next action: Kick off concept phase using existing claude-design-handoff materials.  
Risk: May redesign before auth model is known.

**5. Prioritize Data / Import / Workflow Hardening**  
When to choose: Staff and print shop feedback highlights data quality or import pain.  
Required evidence: Real usage logs from demo + staff interviews.  
Owner: Backend + operations team.  
Next action: Focused hardening sprint on import V2 and print workflows.  
Risk: Less visible to executives.

**6. Pause for Governance / DPO Review**  
When to choose: Strong concerns raised about PDPA, retention, or external data flows.  
Required evidence: DPO meeting scheduled + written questions.  
Owner: DPO + EMS owner.  
Next action: Formal DPO briefing within 1 week.  
Risk: Can stall momentum if not time-boxed.

**7. Do Not Proceed**  
When to choose: Overwhelming negative feedback or fundamental misfit.  
Required evidence: Aggregated feedback + leadership consensus.  
Owner: Project sponsor.  
Next action: Document lessons and archive or pivot.

**Recommended Default Path (based on current evidence)**  
- Immediately send Laravel Auth Contract request (highest leverage).  
- Parallel: Begin light design concept work (low risk).  
- Do not start auth bridge implementation until contract is closed and verified.
