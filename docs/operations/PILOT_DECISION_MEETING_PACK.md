# PILOT_DECISION_MEETING_PACK.md

**Date**: 2026-05-22  
**Purpose**: Complete package for a short decision meeting with project owner, IT, and admin.

---

## 1. Meeting Purpose

To select the pilot runtime target environment and assign owners so that configuration and evidence collection for the EMS controlled pilot can begin immediately.

---

## 2. Decisions Required

1. Choose pilot target environment (Faculty LAN Server, Docker/VM, etc.)
2. Assign IT / Infrastructure owner
3. Assign EMS Deployment owner
4. Confirm network scope and pilot user group
5. Set target pilot start date
6. Confirm DPO reviewer

---

## 3. Recommended Decision Path

- Primary: Faculty LAN Server or Docker/VM
- Use Local Rehearsal only while real target is being prepared
- Cloud only after PDPA review

---

## 4. Open Blockers (as of 2026-05-22)

- Pilot target environment not designated (Critical)
- SECRET_KEY, DATABASE_URL, Backup, DPO sign-off (all depend on target)

---

## 5. Documents to Review in the Meeting

- Executive Pilot Decision Brief
- Pilot Target Decision Package
- Pilot Environment Decision Matrix
- IT Handoff Action Pack
- Pilot Human Action Tracker
- Pilot Launch Sequence

---

## 6. Meeting Agenda (30-45 minutes recommended)

1. Confirm pilot objective and user scope (5 min)
2. Review environment options and recommendation (10 min)
3. Choose target and assign IT owner (10 min)
4. Assign DPO reviewer and system owner (5 min)
5. Confirm timeline and evidence process (5 min)
6. Agree on next actions and owners (5 min)
