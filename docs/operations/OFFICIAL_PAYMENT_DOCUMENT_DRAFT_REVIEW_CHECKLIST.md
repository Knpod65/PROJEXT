# แบบตรวจทานร่างเอกสารเบิกจ่ายค่ากรรมการคุมสอบและกรรมการจ่ายข้อสอบ

**สถานะเอกสาร**: `DRAFT_NOT_AUTHORIZED`
**สถานะ gate ปัจจุบัน**: `PENDING_SUPERVISOR_FINANCE_REVIEW`
**ขอบเขต**: ตรวจทานรูปแบบร่างเอกสารสำหรับภาคเรียน `2/2568` เท่านั้น

> หมายเหตุสำคัญ: แบบตรวจทานนี้ไม่ใช่การอนุมัติจ่ายเงิน ไม่ใช่เอกสารสิทธิ์การเบิกจ่ายขั้นสุดท้าย และไม่ใช่การยืนยันข้อมูลเป็นความจริงขั้นสุดท้าย จนกว่าจะผ่านการตรวจทานจากผู้บังคับบัญชา/การเงินตามกระบวนการที่ได้รับอนุญาต

## 1. ตรวจรูปแบบเอกสาร

- [ ] ตารางร่างมีรูปแบบสอดคล้องกับตัวอย่างเอกสาร `2/2568`
- [ ] ตารางแยกกลุ่มตามวันที่สอบและช่วงเวลาอย่างถูกต้อง
- [ ] ตารางแสดงประเภทวัน เช่น วันธรรมดา / วันหยุด อย่างชัดเจน
- [ ] ตารางแสดงทั้งหมวด `INVIGILATION_COMMITTEE / กรรมการคุมสอบ`
- [ ] ตารางแสดงทั้งหมวด `PAPER_DISTRIBUTION_COMMITTEE / กรรมการจ่ายข้อสอบ`
- [ ] ผลรวมรายแถวและผลรวมทั้งเอกสารอ่านได้ชัดเจน

## 2. ตรวจอัตรา

- [ ] ใช้อัตราเฉพาะภาคเรียน `2/2568`
- [ ] วันจันทร์-ศุกร์ใช้อัตรา `120 THB` ต่อคน/ครั้ง
- [ ] วันเสาร์-อาทิตย์ใช้อัตรา `200 THB` ต่อคน/ครั้ง
- [ ] ไม่ใช้อัตรา active demo/test `300/500` เป็นแหล่งอ้างอิงทางการ
- [ ] ไม่มีการเปลี่ยน active rate ของระบบจากการตรวจทานนี้

## 3. ตรวจข้อมูลกรรมการคุมสอบ

- [ ] จำนวนกรรมการคุมสอบมาจากข้อมูลมอบหมายงานที่เข้าเกณฑ์ร่างเอกสาร
- [ ] รายการที่ถูก block หรือไม่เข้าเกณฑ์ไม่ถูกนับเป็นยอดจ่ายในร่าง
- [ ] กรณีมี warning ต้องตรวจเหตุผลและรายการที่เกี่ยวข้อง
- [ ] ข้อมูลยังเป็นร่างเพื่อการตรวจทาน ไม่ใช่ยอดจ่ายขั้นสุดท้าย

## 4. ตรวจข้อมูลกรรมการจ่ายข้อสอบ

- [ ] จำนวนกรรมการจ่ายข้อสอบเป็นข้อมูล staff-confirmed/manual-confirmed สำหรับร่างเอกสารเท่านั้น
- [ ] ข้อมูลที่กรอกเองไม่ถูกบันทึกถาวรจากการเปิดร่างนี้
- [ ] ช่องวันที่และช่วงเวลาของรายการ manual ตรงกับตารางร่าง
- [ ] รายการ manual ที่ไม่มีแถวกรรมการคุมสอบคู่กันถูกแสดงพร้อม warning เพื่อให้ตรวจทาน
- [ ] แหล่งข้อมูลกรรมการจ่ายข้อสอบอย่างเป็นทางการยังต้องได้รับการ validate แยกต่างหาก

## 5. ตรวจสถานะเอกสาร

- [ ] เห็นสถานะ `DRAFT_NOT_AUTHORIZED`
- [ ] ไม่มีปุ่ม approve / authorize / final payment
- [ ] ไม่มีปุ่มหรือข้อความ official export / PDF / Excel
- [ ] ไม่มีข้อความที่ทำให้เข้าใจว่าเป็นเอกสารเบิกจ่ายขั้นสุดท้าย
- [ ] ไม่มี logic หรือข้อความเกี่ยวกับ teaching workload / Work H / opencourse / coinstruc

## 2026-06-08 ชุดเอกสารสำหรับผู้บังคับบัญชา/ฝ่ายการเงิน

ชุดเอกสารประกอบการตรวจทานที่อ่านง่ายสำหรับผู้ที่ไม่ใช่นักพัฒนาเตรียมไว้แล้ว:

- `docs/operations/SUPERVISOR_FINANCE_REVIEW_PACKAGE_SOURCE_REVIEW.md`
- `docs/operations/EMS_PAYMENT_DOCUMENT_DRAFT_REVIEW_ONE_PAGER.md`
- `docs/operations/SUPERVISOR_FINANCE_REVIEW_DECISION_FORM.md`
- `docs/operations/SUPERVISOR_FINANCE_REVIEW_TALKING_SCRIPT.md`
- `docs/operations/SUPERVISOR_FINANCE_QUICK_REVIEW_CHECKLIST.md`

สถานะ gate ยังเป็น `PENDING_SUPERVISOR_FINANCE_REVIEW` และสถานะเอกสารยังเป็น `DRAFT_NOT_AUTHORIZED` จนกว่าจะมีผลการตรวจทานจริงจากผู้มีอำนาจ

Decision meaning:

- `ACCEPT_DRAFT_FORMAT`: ยอมรับรูปแบบร่างเอกสารเพื่อพิจารณา workflow ร่าง export ในอนาคต แต่ยังไม่ใช่การอนุมัติจ่ายเงิน
- `ACCEPT_WITH_CORRECTIONS`: ยอมรับแนวทางโดยมีรายการแก้ไข ต้องบันทึก backlog และ re-smoke หลังแก้ไข
- `HOLD_PENDING_DATA_SOURCE`: พักไว้ก่อนจนกว่าจะ validate แหล่งข้อมูลกรรมการจ่ายข้อสอบอย่างเป็นทางการ
- `REJECT_DRAFT_FORMAT`: ไม่ยอมรับรูปแบบร่าง ต้อง redesign draft format ก่อนดำเนินการต่อ
