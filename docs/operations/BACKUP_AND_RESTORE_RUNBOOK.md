# Backup and Restore Runbook

**System**: EMS (Exam Management System)
**Last Updated**: 2026-05-22
**Owner**: IT + System Owner + DPO

## 1. Scope of Backup

### What Must Be Backed Up
- **PostgreSQL Database** (primary)
  - All operational tables (schedules, submissions, workload, users, audit, etc.)
  - Governance and PDPA-related tables
- **Uploaded Files / Attachments**
  - Exam papers, submissions, supporting documents
- **Generated Exports**
  - CSV, PDF, and other exports generated by the system
- **Audit Logs**
  - Database audit trail (if stored in DB)
  - Application audit logs
- **Configuration Files (excluding secrets)**
  - Non-sensitive configuration

### What Must NOT Be Backed Up in the Same Location
- `SECRET_KEY`
- Database credentials
- Any other production secrets

## 2. Backup Frequency

| Type                    | Frequency          | Trigger                          | Retention          |
|-------------------------|--------------------|----------------------------------|--------------------|
| Daily Pilot Backup      | Daily at 02:00     | Scheduled                        | 30 days            |
| Pre-Deployment Backup   | Before every deploy| Manual trigger via runbook       | 90 days            |
| Pre-Migration Backup    | Before DB migrations| Manual + automated hook         | 180 days           |
| On-Demand               | As needed          | Security incident, major change  | As decided by DPO  |

## 3. PostgreSQL Backup Commands

### Recommended: Custom Format (recommended for large databases)
```bash
pg_dump \
  -h $DB_HOST \
  -U $DB_USER \
  -d $DB_NAME \
  -Fc \
  -f /backups/ems_$(date +%Y%m%d_%H%M%S).dump
```

### Plain SQL (for smaller instances or quick inspection)
```bash
pg_dump \
  -h $DB_HOST \
  -U $DB_USER \
  -d $DB_NAME \
  --clean --if-exists \
  | gzip > /backups/ems_$(date +%Y%m%d_%H%M%S).sql.gz
```

### Compressed with encryption (recommended for PDPA data)
```bash
pg_dump -Fc -h $DB_HOST -U $DB_USER -d $DB_NAME | \
  gzip | \
  openssl enc -aes-256-cbc -salt -pbkdf2 \
  -pass file:/secure/backup_passphrase.txt \
  -out /backups/ems_$(date +%Y%m%d_%H%M%S).dump.gz.enc
```

## 4. Restore Commands

### From custom format dump
```bash
pg_restore \
  -h $DB_HOST \
  -U $DB_USER \
  -d $DB_NAME \
  --clean --if-exists \
  /backups/ems_YYYYMMDD_HHMMSS.dump
```

### From encrypted backup
```bash
openssl enc -d -aes-256-cbc -pbkdf2 \
  -pass file:/secure/backup_passphrase.txt \
  -in /backups/ems_YYYYMMDD_HHMMSS.dump.gz.enc | \
  gunzip | \
  pg_restore -h $DB_HOST -U $DB_USER -d $DB_NAME --clean --if-exists
```

## 5. Restore Verification Checklist

After every restore, the following **must** be verified:

- [ ] Application starts successfully
- [ ] `/health` or health endpoint returns 200 OK
- [ ] Login works for at least one Admin and one Teacher account
- [ ] Admin Intelligence Dashboard loads without errors
- [ ] Workload Duty Analytics returns data (not empty error)
- [ ] Audit logs are visible and queryable
- [ ] At least one submission and one schedule record can be viewed
- [ ] No critical errors in application logs

**Only after all checks pass** is the restore considered successful.

## 6. PDPA-Sensitive Backup Rules

- All production backups containing personal data **must** be encrypted at rest.
- Backup files must be stored in access-controlled locations (not public S3 buckets).
- Retention period: maximum 365 days unless DPO approves extension.
- Deletion of backups must be logged and approved by DPO when containing personal data.
- No personal data (names, IDs, emails) should appear in backup file names or unencrypted metadata.

## 7. Roles and Responsibilities

| Role              | Responsibility                              | Contact     |
|-------------------|---------------------------------------------|-------------|
| IT / DevOps       | Execute daily and pre-deployment backups    | IT Team     |
| System Owner      | Own backup policy and restore decisions     | EMS Owner   |

## 8. Backup Location & Security

- Primary: Encrypted storage in institutional infrastructure
- Secondary (recommended): Offsite / different availability zone
- Access: Restricted to IT + System Owner + DPO (as needed)

**Never store unencrypted backups containing personal data.**
