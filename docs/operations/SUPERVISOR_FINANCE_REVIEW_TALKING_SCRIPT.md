# สคริปต์อธิบายร่างเอกสารสำหรับผู้บังคับบัญชา/ฝ่ายการเงิน

**English title**: Supervisor / Finance Review Talking Script  
**วันที่จัดทำ**: 2026-06-08  
**สถานะเอกสาร**: `DRAFT_NOT_AUTHORIZED`

## สรุป 3 นาที

1. EMS มีหน้าร่างเอกสารสรุปค่าตอบแทนรูปแบบใกล้เคียงตัวอย่างภาคเรียน `2/2568`
2. เอกสารร่างแสดงทั้ง `กรรมการคุมสอบ` และ `กรรมการจ่ายข้อสอบ`
3. ร่างนี้ใช้อัตรา `120/200` สำหรับ 2/2568 เพื่อให้ตรวจรูปแบบก่อน แต่ยังไม่ใช่การอนุมัติจ่ายเงินจริง
4. อัตรา active `300/500` ในระบบยังถือเป็น demo/test data สำหรับเส้นทางนี้
5. จำนวนกรรมการจ่ายข้อสอบยังต้องยืนยันแหล่งข้อมูลทางการ
6. วันนี้ต้องการให้ผู้ตรวจทานตัดสินใจเรื่องรูปแบบเอกสาร อัตรา แหล่งข้อมูล และขั้นตอนถัดไป

ประโยคที่ควรพูดชัดเจน:

> หน้านี้เป็นร่างเพื่อการตรวจทานเท่านั้น สถานะยังเป็น `DRAFT_NOT_AUTHORIZED` และยังไม่มี approval, final authorization, PDF, Excel หรือ official export

## Walkthrough 10 นาที

1. เปิดภาพรวมสถานะ
   - ระบุว่า UI ผ่านการจัดรูปแบบสำหรับ review แล้ว
   - ชี้สถานะ `DRAFT_NOT_AUTHORIZED`
   - ย้ำว่าไม่มีปุ่มอนุมัติหรือ export ทางการ

2. อธิบายรูปแบบตาราง
   - จัดกลุ่มตามวันที่สอบและช่วงเวลา
   - แสดงประเภทวัน เช่น วันธรรมดา/วันเสาร์-อาทิตย์
   - แยกจำนวนและค่าตอบแทนของสองหมวดหน้าที่
   - แสดงยอดรวมต่อแถวและยอดรวมทั้งเอกสาร

3. อธิบายอัตรา
   - ตัวอย่าง 2/2568 แสดง `120 THB` สำหรับวันจันทร์-ศุกร์
   - ตัวอย่าง 2/2568 แสดง `200 THB` สำหรับวันเสาร์-อาทิตย์
   - ต้องให้ผู้ตรวจทานยืนยันว่าจะใช้อัตรานี้ อัตราอื่น หรือพักไว้ก่อน

4. อธิบายแหล่งข้อมูลกรรมการจ่ายข้อสอบ
   - ขณะนี้ใช้ staff-confirmed/manual-confirmed สำหรับร่างเท่านั้น
   - ยังไม่ถือว่าเป็น authoritative payable source
   - ต้องให้ผู้ตรวจทานเลือกแหล่งข้อมูลหรือสั่ง validate แหล่งข้อมูลก่อน

5. อธิบาย check-in / no-show / absence
   - Check-in เป็นข้อมูล reconciliation หลังปฏิบัติหน้าที่ ไม่ใช่ pre-disbursement gate ใน pass นี้
   - กรณีขาดปฏิบัติหน้าที่หรือ no-show ต้องมีคำอธิบายและหลักฐานในขั้น reconciliation
   - หากพบรายการที่จ่ายล่วงหน้าไม่ถูกต้อง ต้องจัดการ refund/offset ตามกติกาที่อนุมัติในอนาคต

6. ปิดด้วยแบบฟอร์มตัดสินใจ
   - ขอให้เลือก decision code ที่ตรงกับผลตรวจทาน
   - บันทึกชื่อ ตำแหน่ง วันที่ และหมายเหตุ
   - ถ้ายังไม่ยืนยัน rate หรือ paper source ให้เลือก hold/validate ก่อน

## คำถามที่คาดว่าจะเจอและคำตอบที่ปลอดภัย

**ถาม: เอกสารนี้อนุมัติจ่ายเงินได้เลยหรือไม่?**  
ตอบ: ไม่ได้ เอกสารนี้เป็นร่างเพื่อ review เท่านั้น สถานะยังเป็น `DRAFT_NOT_AUTHORIZED`

**ถาม: กด export PDF หรือ Excel ได้หรือยัง?**  
ตอบ: ยังไม่ได้ ยังไม่มี official PDF/Excel/export ใน pass นี้

**ถาม: ทำไมไม่ใช้ active rate `300/500`?**  
ตอบ: สำหรับเส้นทางร่าง 2/2568 นี้ active `300/500` ถูกระบุเป็น demo/test data เท่านั้น ต้องรอผู้ตรวจทานยืนยัน rate ที่ใช้จริง

**ถาม: จำนวนกรรมการจ่ายข้อสอบมาจากไหน?**  
ตอบ: ตอนนี้ใช้ staff-confirmed/manual-confirmed เพื่อดูร่างเอกสารเท่านั้น ยังต้องยืนยัน authoritative source ก่อนถือเป็นข้อมูลทางการ

**ถาม: ถ้ามีคนไม่มาปฏิบัติหน้าที่จะทำอย่างไร?**  
ตอบ: ต้องใช้ขั้น reconciliation หลังปฏิบัติหน้าที่ พร้อมเหตุผล/หลักฐาน และถ้ามีผลต่อเงินต้องจัดการ refund/offset ตามกติกาที่อนุมัติภายหลัง

**ถาม: ถ้ารูปแบบนี้ยอมรับได้ ขั้นต่อไปคืออะไร?**  
ตอบ: อาจออกแบบ draft export workflow ใน pass ถัดไปได้ แต่ยังไม่ใช่ final authorization หรือ payment approval
