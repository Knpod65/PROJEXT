# EMS Thai PDF Font Embedding Contract

Date: 2026-06-16

## Contract

PDF generators must register Thai-capable fonts from locally installed font files only. No font binaries are committed to the repository.

`backend/services/thai_export_service.py` exposes `register_reportlab_thai_fonts()`, which returns:

- `normal_name`
- `bold_name`
- `normal_path`
- `bold_path`
- `embedded`

When a local font candidate is found, ReportLab registers it and generated PDFs can embed/subset that font according to ReportLab behavior. When no local Thai font is available, the helper returns Helvetica fallback with `embedded=false`. This fallback is explicit and must not be represented as Thai PDF visual acceptance.

## Current Callers

- Schedule PDF export.
- Workload summary PDF export.
- Operational document PDFs.
- Exam PDF header stamping.

## Validation Rule

Automated validation may assert the helper contract and generated bytes. Full Thai visual acceptance for PDF requires a machine with a licensed/installed Thai font and a rendered PDF inspection. This pass does not commit font assets.
