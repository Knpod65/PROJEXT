# Troubleshooting Guide

## Purpose

This guide explains how to respond when EMS does not look right.

It focuses on symptoms, likely causes, fixes, and escalation paths so users do not have to guess.

## What To Use It For

- Permission confusion
- Missing data
- Dashboard confusion
- Alert misunderstanding
- Governance misunderstanding

## Standard Response Pattern

1. Identify the symptom.
2. Check whether the issue is role-based, data-based, or workflow-based.
3. Confirm the owning page or service.
4. Apply the smallest safe fix.
5. Escalate when the issue affects publication, safety, or audit integrity.
