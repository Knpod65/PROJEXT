# Teacher Manual

## Purpose

Teachers use EMS to submit exam work, understand personal workload, and follow assigned responsibilities.

The guide should make the teacher path feel simple and safe, not administrative.

## Daily Workflow

1. Open the role dashboard or my work page.
2. Review assigned exams, deadlines, and submission status.
3. Check workload signals if the role can see them.
4. Submit required exam material or respond to outstanding action items.
5. Confirm whether any escalation is needed for missing or conflicting assignments.

## Most Important Pages

- Dashboard
- My Exam Work
- Submissions
- Schedule
- Workload view if available for the role

## What Teachers Should Watch For

- Pending submission deadlines
- Missing assignment details
- Conflicts between assigned duties and availability
- Feedback that indicates a workflow did not complete correctly

## Common Mistakes

- Assuming a task is complete because the page loaded successfully
- Ignoring a warning banner about a missing submission or validation failure
- Confusing personal workload with department-wide analytics

## Best Practices

- Check deadlines early.
- Read the status text before opening a detailed page.
- Escalate to the right operational contact if an assignment does not match reality.
- Use the simplest page that solves the task.

## Escalation Flow

If the problem is about an assignment, deadline, or submission state, escalate through the exam workflow contact path.

If the problem is about access or role mismatch, contact administration support.

## Governance Responsibility

Teachers are responsible for accurate submission and honest task completion.

They should treat system status as an operational signal, not as a replacement for checking the underlying instruction.
