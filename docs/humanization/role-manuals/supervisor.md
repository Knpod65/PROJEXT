# Supervisor Manual

## Purpose

Supervisors oversee an operational slice of EMS and make sure the day runs without avoidable surprises.

They often bridge the gap between staff execution and admin oversight.

## Daily Workflow

1. Check the local dashboard for exceptions and coverage gaps.
2. Review workflow items that need verification.
3. Monitor room, distribution, or attendance issues in real time.
4. Confirm that staff actions match the expected operational path.
5. Escalate any issue that cannot be resolved within the local scope.

## Most Important Pages

- Dashboard
- Operational Health
- Checkins
- Room Attendance
- Governance Cockpit if the supervisor can see escalation items
- Workload Duty Analytics if relevant to the role scope

## What Supervisors Should Watch For

- Missing coverage
- Delay patterns
- Repeated workflow failures
- Misaligned assignments
- Unexpected escalation volume

## Common Mistakes

- Solving a policy issue as if it were only an operational one
- Staying at summary level when the workflow page already shows the cause
- Escalating too late because the problem looked small at first

## Best Practices

- Check the trend, not just the current value.
- Verify the workflow state before calling an issue resolved.
- Keep the local response small and fast.
- Escalate as soon as the issue crosses the supervisor’s scope.

## Governance Responsibility

Supervisors are responsible for keeping the local operation truthful and controlled.

Their role is to detect drift early and ensure the right handoff happens at the right time.
