# Dashboard Interpretation Guides

## Purpose

Dashboard guides explain what each screen means in operational terms.

They should not describe charts only as technical widgets. They should explain how to read risk, urgency, readiness, and next action.

## Standard Guide Structure

Each dashboard guide should include:

1. What the dashboard is for
2. Who should use it
3. What the top signals mean
4. What to ignore
5. What counts as urgent
6. What to do next
7. When to escalate
8. What governance rule it supports

## Interpretation Rules

- Explain colors as meaning, not decoration.
- Explain trends as movement, not statistics.
- Explain thresholds as action triggers, not abstract numbers.
- Explain comparisons as operational context, not ranking for its own sake.
