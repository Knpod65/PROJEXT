# Screenshot Atlas

## Purpose

The atlas turns EMS screens into annotated visual teaching material.

Each screenshot should show the important areas only, with labels that explain what the user should notice and why it matters.

## Atlas Rules

- Start with the highest-traffic pages.
- Capture the screen only after the written explanation exists.
- Annotate what matters first, not every visible element.
- Explain what to ignore when that will reduce confusion.

## Suggested Sections

- [Major Pages](major-pages.md)
- Dashboard pages
- Operations pages
- Governance pages
- Analytics pages
- Workflow pages

## Status

The atlas is currently a structure and annotation system. The first screenshot capture pass should be built from the major-pages list once visuals are collected.

## Outcome

Users should be able to open a screenshot and immediately understand the purpose of the page, the urgent signal, and the next action.
