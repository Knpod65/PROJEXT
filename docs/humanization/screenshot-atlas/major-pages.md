# Screenshot Atlas: Major Pages

## Purpose

This file defines the first screenshot pass for the humanization system.

The atlas should start with the pages users visit most often and the pages that explain risk, readiness, and action.

## Per-Screenshot Annotation Plan

For each page, capture:

1. The page title and role context
2. The main summary signal
3. The one or two areas the user should inspect first
4. The action control or next-step button
5. Any warning, blocker, or escalation state

## Callout Rules

- Use one callout for the main purpose of the page
- Use one callout for the primary action
- Use one callout for the urgent warning if present
- Avoid labeling every control on the screen

## What Matters

- The user’s first decision point
- The signal that changes the next action
- The role boundary for the page
- The most important warning or success state

## Ignore This For Now

- Decorative layout elements
- Secondary filters that do not change the main action
- Non-essential metadata when it does not affect the workflow

## Beginner Notes

- Explain the screen in one sentence before the visual callouts
- Keep the first screenshot wide enough to show context
- Highlight only what the user must notice first

## Advanced Notes

- Include state changes and edge-case warnings
- Show the difference between summary views and detail views
- Add trace or audit notes when a decision depends on evidence

## Placeholder Convention

Each screenshot should later be named by page, role, and sequence number.

Example:

- `dashboard-admin-01.png`
- `governance-cockpit-admin-02.png`
- `teacher-submission-01.png`
