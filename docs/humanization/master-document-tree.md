# Master Document Tree

## Purpose

This tree is the publication plan for making EMS understandable to different audiences.

It separates the documentation by how people use the platform, not by how the codebase is organized.

## Publication Order

1. Master cognitive map
2. Humanization strategy
3. Role manual index
4. Dashboard guide index
5. The most-used role manuals
6. The most-used dashboard guides
7. Visual journeys for the highest-frequency workflows
8. Troubleshooting and playbooks
9. Screenshot atlas
10. Training decks and export packaging

## Priority Build Order

Start with the documents that reduce uncertainty fastest:

1. Admin manual
2. Teacher manual
3. Staff manual
4. Dashboard interpretation guide
5. Governance cockpit guide
6. Workload analytics guide
7. Operational health guide

## Notes

- This tree is intentionally role-led and workflow-led.
- The same dashboard may appear in multiple manuals if the interpretation changes by role.
- Screenshot pages should be added after the written guide exists, not before it.
