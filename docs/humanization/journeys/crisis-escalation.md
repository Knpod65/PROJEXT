# Crisis Escalation Journey

## Operational Purpose

This journey shows what happens when a problem cannot be handled at the local level and must be escalated quickly.

## Expected Mindset

The user should prioritize speed, clarity, and ownership.

## Step-by-Step Flow

1. Identify the problem.
2. Check whether it affects active operations, publication, or safety.
3. Confirm the owner of the affected workflow.
4. Escalate to the correct role immediately.
5. Record the reason for escalation.
6. Follow up until the issue is acknowledged.

## Screenshot Placeholders

- Screenshot 1: alert or failure signal
- Screenshot 2: escalation action
- Screenshot 3: owner confirmation
- Screenshot 4: resolution or handoff status

## Annotation Instructions

- Highlight the urgent alert
- Circle the escalation action
- Label the owner and priority
- Mark any unresolved blocker

## Governance Implications

Crisis escalation must preserve traceability while moving fast.

## Stress Points

- Ambiguous ownership
- Multiple simultaneous alerts
- Delayed acknowledgment

## Common Errors

- Waiting too long before escalating
- Escalating to the wrong owner
- Failing to record what was observed

## Recovery Path

- Re-check the problem scope
- Confirm ownership
- Escalate again if the first path failed
