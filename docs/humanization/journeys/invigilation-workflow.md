# Invigilation Workflow Journey

## Operational Purpose

This journey shows how staff or assigned invigilators move from room readiness to active invigilation and completion.

## Expected Mindset

The user should be alert, calm, and focused on room stability.

## Step-by-Step Flow

1. Check the assigned room and time.
2. Confirm presence, materials, and readiness.
3. Start the invigilation period.
4. Monitor attendance and room conditions.
5. Record any change or issue.
6. Close the session when the exam ends.
7. Escalate if the room becomes unstable.

## Screenshot Placeholders

- Screenshot 1: assignment and readiness view
- Screenshot 2: live invigilation state
- Screenshot 3: attendance or incident update
- Screenshot 4: end-of-session confirmation

## Annotation Instructions

- Highlight room name and time
- Mark the live state indicator
- Label incident or warning areas
- Show the completion action clearly

## Governance Implications

Invigilation creates direct evidence that the exam was supervised correctly.

## Stress Points

- Missing invigilator
- Room disruption
- Attendance mismatch
- Timing confusion

## Common Errors

- Starting without checking the live room state
- Ignoring a warning that needs action
- Closing the session before confirming completion

## Recovery Path

- Verify the room status
- Re-check the assigned staff and materials
- Escalate if the session cannot continue safely
